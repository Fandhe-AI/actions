# delegation（調査・設計フェーズの委譲）

main コンテキストの消費を抑えるため、調査・設計は subagent へ委譲する。main は
「何を調べるか」を決め、結果を統合して意思決定することに集中する。

## 原則

- 複数ファイルを横断して読む調査は `explorer` に委譲し、結論だけ受け取る
- main 自身は大量のファイル全文を読み込まない（要点・所在・差分を受け取る）
- 設計判断・トレードオフの最終決定は main が行う

## パスベース切り替え（調査対象 → 委譲先）

| 調査対象 | 委譲先 |
|---|---|
| `*/action.yml`・`*/README.md` の構造・入出力調査 | `explorer` |
| 変更差分の品質確認 | `reviewer` |
| セキュリティ観点の確認 | `security-auditor` |

## 委譲しない場合

- 単一ファイルの 1 箇所をピンポイントで確認するだけのとき
- すでに会話内で判明している事実の再確認
