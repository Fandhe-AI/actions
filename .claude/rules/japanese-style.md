# japanese-style（日本語出力スタイル）

ユーザーとのやりとり・ドキュメント・コミットメッセージ・コメントは原則 **日本語** で書く。

## 方針

- 説明・報告・提案は日本語。簡潔・具体的に書く
- 技術用語・コード識別子・コマンド・ファイル名は原語のまま（無理に訳さない）
- 半角英数と日本語の間は自然な体裁を保つ（過剰なスペースは不要）
- アクセント付き文字・特殊文字は正書法どおりに保持する（ASCII 代替にしない）

## ドキュメント

- README / CLAUDE.md は日本語で記述する
- 表・箇条書きを活用して見通しよくする

## コミット / PR

- Conventional Commits の subject は日本語可（`conventional-commits.md` 参照）
- PR 本文（Summary / Test plan）も日本語で記述する
