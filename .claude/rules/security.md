# security（OWASP Top 10 + GitHub Actions 特有）

アクション実装・PR 作成時は以下を必須でチェックする。問題があれば作業を中断して警告する。

## GitHub Actions 特有のリスク

### コマンドインジェクション
`${{ inputs.* }}` / `${{ github.event.* }}` 等のテンプレート式を `run:` 本文へ直接
埋め込むと、悪意ある入力（バッククォート・`$(...)` 等）がシェルで実行されうる。
**必ず `env:` 経由でシェル変数に渡し**、`"$VAR"` として参照する。

### workflow command 注入
`::notice::` / `::error::` / `::set-output::` などの workflow command 行に
ユーザー入力をそのまま展開すると、任意の workflow command を注入されうる。
ユーザー由来の値は**別の `echo` 行**に分離して出力する。

### トークンの取り扱い
- remote URL / git config にトークンを埋め込んだら `trap ... EXIT` で**確実に後始末**する
- トークンをログ（`echo` / `set -x`）に出さない
- 必要最小限の権限のトークンを要求する（`GITHUB_TOKEN` で足りるなら PAT を要求しない）

## 汎用（OWASP Top 10）

- **シークレット混入**: API キー・トークン・パスワード・`.env` をハードコード/コミットしない
- **入力バリデーション**: 信頼できない入力の検証欠如、eval 的な利用
- **認証・認可**: 権限チェックの漏れ
- **XSS / インジェクション**: 出力先に応じたエスケープ

## 運用

- PR 作成前に `security-auditor` で監査する
- シークレットの混入を検出したらコミット/PR を中止する
